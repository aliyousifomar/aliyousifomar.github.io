Integrating RHEL Servers with Active Directory for Centralized Identity Management

Integrating RHEL (Red Hat Enterprise Linux) servers with your organization's Active Directory (AD) streamlines user management and enhances security. Below is a step-by-step guide, formatted for clarity, to help you join RHEL systems to AD using standard tools and best practices.

  • Prerequisites:
    • Ensure your RHEL server can resolve AD domain controllers via DNS.
    • Have domain admin credentials ready.
    • Update your system packages.
  • Install Required Packages: 
    sudo yum install sssd-ad sssd-tools realmd adcli oddjob oddjob-mkhomedir samba-common-tools
  • Join the Domain: 
    sudo realm join --user=Administrator corp.example.com
    • Replace corp.example.com with your actual domain.
    • Enter the AD admin password when prompted.
  • Prepare the AD security groups planned to get administrative access to the server
    • Ensure the AD group is created and populated with the necessary users.
    • Some organizations adopt a leading dash (-) with security groups naming, This has caused issues when I tried to incorporate groups with a leading dash, instead create groups in the following fashion rhel-admins in AD.
  • Configure SSSD:
    • Edit /etc/sssd/sssd.conf to reflect your system's setting
    
[sssd]
domains	= corp.example.com
config_file_version	= 2
services = nss, pam, ssh
#manually added
#default_domain_suffix = True
[domain/corp.example.com]
ad_domain = corp.example.com
default_shell = /bin/bash
krb5_store_password_if_offline = True
cache_credentials = True
krb5_realm = corp.example.com
realmd_tags = manages-system joined-with-adcli
id_provider = ad
fallback_homedir = /home/%u
ad_domain = corp.example.com
use_fully_qualified_names = False
ldap_id_mapping	= True
access_provider	= ad
auth_provider = ad
ad_gpo_ignore_unreadable = True
#Allow only specific groups
ad_access_filter = (|(memberOf=CN=Domain Admins,CN=Users,DC=corp,DC=example,DC=com)(memberOf=CN=rhel-admins,OU=AP,OU=rME,OU=Server,OU=Restricted_Groups,OU=Admin,OU=_example,DC=example,DC=corp,DC=com))
  • Restart SSSD Service: 
    sudo systemctl restart sssd
  • This will ensure each user will enter username only without the reference to domain (such as username@domain.com)
  • Each time a new user login completed, a home directory will be created in the following fashion 
    /home/username
  • Set Permissions:
    • Ensure the /etc/sssd/sssd.conf file has the correct permissions:
      • sudo chmod 600 /etc/sssd/sssd.conf
