Deploying Red Hat Satellite for centralized RHEL Provisioning and Management in VMware

  1. Deploying the Red Hat Satellite VM
    • Provision a dedicated VM
      For Red Hat Satellite on your VMware environment. Ensure the VM meets Red Hats recommended requirements for CPU, memory, storage, and network. Refer to the official Red Hat documentation for specific requirements.Red Hat Satellite Requirements
    • Install Red Hat Enterprise Linux on the VM, then install and configure Red Hat Satellite.
    • Register the Satellite server with Red Hat Subscription Management and apply the necessary entitlements.
  2. Integrating Satellite with VMware vCenter
    • In the Satellite web interface, go to Infrastructure > Compute Resources and add your VMware vCenter as a compute resource.
      Note:Make sure you have set up a service account within VCenter with the necessary priviledges.
    • Enter the vCenter server address,service account credentials, and select the appropriate data center(s).
    • Configure network, storage, and template settings to enable Satellite to provision VMs directly in vCenter.
  3. Automatic Provisioning of RHEL Servers
    • Define compute profiles in Satellite to standardize VM hardware settings (CPU, RAM, disk, network).
    • Create host groups that bundle OS version, activation keys, content views, and compute profiles for consistent deployments.
    • When a new RHEL VM is needed, provision it from Satellite; the system will orchestrate VM creation in vCenter, OS deployment, and registration with Satellite.
  4. Configuring Satellite as a Local Repository
    • In Satellite, synchronize content repositories from the Red Hat CDN to create a local mirror of RHEL packages and updates.
    • Organize content into lifecycle environments (e.g., development, QA, production) and content views to control which updates are available to which systems.
    • - Register all managed RHEL servers to Satellite so they receive updates and packages from the local repository, reducing external bandwidth usage and improving control.
  5. Patching and Security Updates
    • Use Satellite patch management features to schedule and deploy security updates and patches to registered hosts.
    • Monitor compliance and update status through Satellite reporting tools.
    • Push or schedule updates based on host group, environment, or custom criteria, ensuring all systems remain secure and up to date.
